Anatomy of a Script
Variables

A variable is like a container. We can put different things (values) into the container. Some important aspects of variables:

Name and Value

A variable has a name and a value. We use the name to refer to the variable. The value is whatever is in the container!

Type

All variables have a type. If a variable is a container, then the type is what kind of container it is - what kind of things it can hold. Some common types:

  • Integer - A number without a decimal point
  • Float - A number with a decimal point
  • Boolean - either true or false
  • String - can contain letters and numbers

Public/Private

Variables also have a private/public/protected status. By default, a variable is private which means it can't be accessed outside of the script it is in. A public variable can be changed by other scripts and can be changed in the Inspector.

Declaring and Initializing a variable

Declaring a variable means giving it a name, type, and public/private status. To declare a public boolean called “jumping”, you'd write:

public bool jumping;

Initializing a variable means giving it a starting value. We usually do this in the Start function:

jumping = false;

Variable Scope

The scope of a variable is where it exists. A global variable can be used by all the functions in a script. A local variable only exists to part of a script. This depends on where it is declared. A variable declared at the beginning of the script, before all the functions, is global. A variable declared inside of a function is local.

Functions

Let's talk about the anatomy of a function! A function has basically 5 parts:

  • Name - the name of the function!
  • start - the bracket at the beginning of the function's code.
  • End - the bracket at the end of the function's code.
  • Parameters - things that the function needs to run its code.
  • Return type - what the function spits out when it is done running.
