Your User Interface. This is the bread and butter of gaming.

From here, your player is able to see everything. You can control how much is on the screen, how cluttered or spartan you want.

Why do we play games?

Sometimes it's a storyline that drives us as players, sometimes it's the experience. While there are a million different reasons to play video games, competition is a great motivator. Tracking points is an immersive way to provide incentive to a player to interact in your world.

Our players need a reason to pick up the collectable that we just designed. Let's give them one - points! Think of a Mario game. In a Mario game there are often coins all around that the player can pick up. You'll probably make a bunch of copies of this prize and distribute them all around. Before we make a pre-fab (a pre-fabricated model that does not require us to code it every single time), we want to make sure it works. We're going to make use a new tool here called the Unity UI.

UI stands for User Interface.

UI covers things such as the menus a player might go through and the HUD (heads-up display). A HUD shows the player information on the screen while the player plays the game. Returning to the Mario example, the HUD in Super Mario 64 shows Mario's lives, his health, his coins, and his stars.

We're going to add a HUD element that shows the player's score.

When building your interface, you have options.

You can make it as simple as Super Mario Bros

Or as complicated as an MMORPG (Massive Multiplayer Online Role Playing Game)

Think about every thing you add to the interface. Make sure it has a distinct use and the interface isn't too cluttered.

Making the UI

We're going to make a new GameObject now - text that shows the score. It's been a while since we've done this. Go to the Hierarchy and hit Create at the top.

Create → UI → Text.

This will create some new objects for us. Let's take a look at them.

You'll see a Canvas, with the text object in it. You'll also see a new object called an EventSystem. These are elements that are necessary for Unity to make the UI.

This is your preliminary UI

Canvas - this big white box. The Canvas has all of the UI elements on it, and it displays them on the screen when the player plays the game.

EventSystem - a manager that makes sure that UI elements and other objects in your scene work together well. We don't have to do anything with it right now, so just leave it alone.

First of all, let's rename our text. We might add more texts later so we want to make sure that we can tell which one this is. You can do this in the Inspector at the top. Name it ScoreText.

Now we want to reposition it. Double-click its name in the Hierarchy to frame-in on it. It's on the Canvas, but it may not be where you want it to be.

Worldbuilding Tip:

Traditionally, a HUD is in a corner of the screen. Think of a place that out of the main view of the character, so that the game is still immersive, but still very interactive.


You can move it using its Rect Transform in the Inspector.

Click on the target-looking thing on the left to move it around. To move it, click the area of your choice and hold SHIFT and ALT while clicking it.

Now your text will be in the place of your choosing. Let's look at the ScoreText's other components in the Inspector. The major one is the Text (Script) component. This lets us format the text. You can make changes to the font, font-size, and color there.

To see what it will look like in-game, go to the Game tab above the Scene view.

The Text component also allows us to change what the text says. But we want the text to update depending on the player's score! We're going to have to use a script. We're going to create a new script called the HUD_Manager.

HUD_Manager

Go to the PlayerController GameObject and click Add Component in the Inspector.

New Script → HUD_Manager

Open it up. Let's get started.

Score Variable

We're going to have to track the player's score. We're going to make a variable to do this. In order to use a variable in your script, you have to declare it. Unity needs to know that the variable exists! We'll want to do this before any functions. Go before the Start function and write:

int score;

There are two parts here - the type of the variable and the name of the variable. Now Unity knows that when we write score, we are referring to this integer variable! Now we can use it.

Look at the comment above the Start function. It says we use this for initialization. We're going to use the Start function to initialize the variable. This means to give it a starting value.

What do we want the player's score to start at?

Linking the UI

Now we want to use this variable to update the UI. In order to do that, we need to add a new namespace. Remember those lines beginning with using at the top of the script? Those allow us to access certain prewritten functions. We're going to add one for the UI functions:

using UnityEngine.UI;

This will allow us to link our HUD_Manager script to our ScoreText object. First, we need to declare it, like a variable.

Text ScoreText;

This is a variable of type Text, that we're calling ScoreText. This does not make the connection for Unity; we have to tell it where to find the ScoreText. We're going to make it set up this connection in the Start function.


Great! Now our two variables are initialized - the score and the text that will display it. Next, we need to code the text to show the score!

Challenge Mode!

This line makes the ScoreText show the score. Where do we put it to show 0 points at the start of the game? Put it there.

ScoreText.text = "Score:" + score;


Increasing the Score: Framework

Time to write our first custom function - one that will update the score! Functions can have any name, but it's best to use a descriptive one. We'll start it right after the Update function.

void UpdateScore(){

}

We can set custom parameters, too. We declare the type and name of the parameter between the parentheses. We'll use an integer amount to change the score by.

void UpdateScore(int amount){

}

Now, we just have to increase the score by the amount!

void UpdateScore(int amount){

score = score + amount;

}

Challenge Mode!

We need one more line in this function. Figure out what we forgot and add it in!


Increasing the Score: It's happening!

We wrote a function in the HUD_Manager script that is able to update the score. We never call it, though, so the score never increases.

When do we want to increase the score? When the collectable is touched! We're not checking that in the HUD_Manager script though. It's in the CollectableScript. We need to call the HUD_Manager's UpdateScore function from the CollectableScript. First, we need to make the UpdateScore function public. Something that is public can be accessed by other scripts. This goes before the return type:

void UpdateScore(int amount){

// your code

}

Connecting to the HUD_Manager script.

First, the CollectableScript has to find the HUD_Manager script. We need to declare and initialize this connection.

HUD_Manager hudmanager;

We're saying that we're going to use a HUD_Manager and we're going to refer to it as hudmanager in this script. To keep it straight, any time we link one script to another in this class we're going to use CamelCase for the class name and all lowercase for the name.

Now we have to initialize the variable - let it know where we can find the HUD_Manager we're going to use.
Challenge Mode!

Where do we put this line?:

hudmanager = GameObject.Find(“PlayerController”).GetComponent <HUD_Manager>();

Remember that we are initializing this connection.


Challenge Mode!

We can now call the function! To call it, we just write the function name.

Where do we put this line:

hudmanager.UpdateScore();

Go ahead and put it where you think it goes! When do we want to increase the score?


Finishing touches

Good work. There's a problem, though. How much will this increase the score by?

Unity doesn't know. The UpdateScore function needed a parameter - the amount! Declare a new variable in the CollectableScript. Call it “points”. We're going to make this variable public. Public variables can be adjusted in the Inspector - this will make it easy to change the number of points on the fly.

We're going to use the points variable as the “amount” parameter when we call the UpdateScore script. What type does it need to be? Fill in the blank when you write your code:

public ____ points;

Cool! Unlike the score, we don't want to initialize this variable in the code. If we do that, it's hard-coded in - we'd need to go change it in the script if we ever wanted to change the amount of points. Instead, we'll set it in the Inspector. Go back to the Unity editor and click on the collectable to go to the Inspector. You should see the Points variable on the CollectableScript component. Fill in the number of points you want the collectable to give you!

Now, all we have to do is add the points variable to the function call:

hudmanager.UpdateScore(points);
