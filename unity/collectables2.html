Rotating Collectables

Open the CollectableScript

We want our collectable to always be rotating, so we can use the Update() function. The Update function runs every frame!

void Update(){

}

In the Update function, we want to access the rotation of the collectable.

Remember that the component that controls the position, scale, and rotation of a GameObject is the transform. We want to change the rotate property of the transform, so we write:

transform.Rotate();

Now when we code we can't just put the collectables' name in there and expect the script to know what we want to rotate, so we are going to have to rename it.

A Vector3 is basically an arrow in 3D space. We're using that arrow as a force, pushing the rotation in the direction of the arrow. It has 3 values: (x, y, z). If we want it to rotate on the y-axis and not the others, we put a value into that slot.

transform.Rotate (new Vector3 (0, 0, 60));

The Update function should look like this:

Go ahead and save that and run the game. You'll notice that it's rotating way too fast! Remember that we are using the Update function - it's called once per frame. Games run at many frames every second, so it is rotating that much many times per second. We want to rotate every second instead. To do this, we multiply the current rotation by something called time.deltaTime. Multiplying by this converts things from frames into seconds.
