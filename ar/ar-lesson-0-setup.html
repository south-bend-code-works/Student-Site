Setup the AR environment

 AR  Lesson 0
In this lesson you will:
  1. Setup Unity to create mobile AR apps.
  2. Learn about ARCore, the SDK for Android AR Development.
  3. Make your first AR app for Android!

Background Info

AR works by detecting planes and other identifiable points/objects in reality, and then allowing the user to create virtual objects and place them in relation to the real objects.

Helpful Links

Docs: AR Core documentation.

Project Setup

Download ARCore SDK from GitHub.
Scroll down to Assets, download the unitypackage.

Open a new Unity 3D project.

Import ARCore SDK into project as a new package.
Assets -> Import Package -> Custom Package

Find the unitypackage you just downloaded.
Select all, then press import.

Inside Window -> Package Manager, install Multiplayer HLAPI and XR Legacy Input Helpers (install the latest versions - which may be newer than those pictured).

Create an empty game object called ARCore Device. Reset the transform to 0,0,0 for position and rotation.

Add the script AR Core Session to ARCore Device.

Find the Session Config and Camera Config Filter inside Assets -> GoogleARCore -> Configurations.

Open the Session Config by double clicking, and set the Camera Focus Mode to Auto Focus.

Drag the Session Config and Camera Config Filter to their respective boxes on the ARCore Session.

Make sure the camera is set to Back Facing.

Drag Main Camera to be a child of ARCore Device.

Set the transform for MainCamera to be 0,0,0.

Set the Clear Flags to be Solid Color.

Add a Tracked Pose Driver script to the MainCamera.

Set the Pose Source to Color Camera.

Set the Update Type to Update.

Enable Use Relative Transform.

Add an AR Core Background Renderer to the MainCamera.

Set the Background Material to ARBackground Material (from Assets -> GoogleARCore -> SDK -> Materials).

Building to Android Device

Computer Steps

In Unity, open Build settings.

Add Open Scenes to Scenes in Build.

Select Android Platform and click Switch Platform (this will take a few minutes).

Open Player Settings.

Set the Company and Product names to whatever you’d like them to be.

Open Other Settings.

Enable Auto Graphics API (this will take a few minutes).

Disable Multi-thread Rendering.

Set the minimum API level to Android 7.0 (SDK 24).

Open XR Settings.

Enable ARCore Supported.

Android Device Steps

Enable Developer Settings (should already be done).

Enable USB Debugging (should already be done).

Computer Steps

Plug in Android via USB.
On the tablet, allow the computer to access data.

Close Player settings

Click Build and Run in the Build Settings dialog.

Choose where to save the .apk and name it appropriately.
The location is not very important. You can leave it in your Unity project folder.
You may need to allow data to be transferred from the Android to the computer when the dialog box appears.
The app will take a few minutes to build and run.

Android Steps

Test your app.
Click to allow the app to record pictures and videos.
Right now you should just see the camera when you open the app.
Lesson complete! Good job!
