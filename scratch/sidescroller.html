Part of what makes this game so much fun, and pretty unique, is the side scrolling terrain that Mario walks on.

It looks like it goes on forever, but in fact you will see there are several sprites that are different sizes and shapes that make up what Mario walks on.

Our first step will be to program these sprites.

Creating a Hitbox

A hitbox is a box used in video games to let the computer know the character has been hit.

We will be using a hitbox in our project, this is an easier way to let the computer know where our sprite is that's more efficent and runs smoother.

Our hit box will be four of the T like shapes.

Click on the first one, topHitTest

We'll start out simple, add a when green flag clicked, a set size to __%, and a set color effect to 0

For our set size to __% block, I changed the size to 50%. We want it to the be same size, or close to our sprite.

But wait, we don't want these big ugly lines to show all the time through our game, let's fix that. Take our set color effect to 0, change colorin our dropdown to ghost, and make it 100 instead of 0.

Now let's create our hitbox, snag a go to player block, then an if/then/else block.

Within our if conditional, we'll put a touching color block, then change it to the color of our terrain.

Let's finish with our conditional, but first we need to make a variable. This will tell the computer that our character is still scrolling through the terrain.

Name this new variable hitTest.

Take your fancy new variable and place it within our if/then/else block twice, the first time we'll set it to 1 but the second time we'll set it to 0.

That way the computer knows if our character isn't touching the floor color since we just created our fancy new hitbox.

But hold on, we're missing one more thing. Right now the computer will only test our hitbox once to see if we're touching the terrain. We need to add a forever around our if/then/else block.
