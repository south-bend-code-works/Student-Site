Lesson 5 - Errors

In this lesson you will:
  1. Learn how to troubleshoot
  2. Learn how to spot errors before they turn into a mess.
Background Info

There are different types of errors, and each one tells you something different about what is wrong. It will help to learn about all the different kinds of errors you will encounter.

Being able to deal with and fix errors is a huge part of programming! Almost nothing you do will work right the first time without errors!

Syntax Errors

You have almost certainly encountered several of these errors already while coding! These are very common while you are learning to code in Python.

Syntax errors will always have a line number, and a little carat ^ pointing to where the error is. Look there and try to figure out what you are missing.

Often, you'll be missing a colon, or your indentation will be off. Sometimes you'll need a comma between parameters, or you may have left off a parenthesis.

EOL while scanning string literal is also common - it means that something is wrong with your string. In this case the quotation marks don't match.

Look carefully at the error message and go back to previous examples of code that works to find your error!

Exceptions

Exceptions are a different type of error. Instead of typing something wrong, there is something logically wrong. Here are three common types of Exceptions.

ZeroDivisionError

Dividing by zero is never allowed.

TypeError

Here we got an error because we tried to concatenate a string and an integer - which is not allowed because they are different data types. Notice how after TypeError: the error is explained.

NameError

This error happens when you don't properly define variables or reference variables that don't exist.

Watch out for misspellings!
