Book of Cain - Diablo III

Simple items like a book, a weapon, or simply a block are good collectables. They tell the player that they need said item for an upcoming event, and can inspire excitement especially if the item is really rare.

From here on out, we will be referring to the object as a collectable.

Download the model and place it in our scene.

Rename it PointsCollectable by right clicking in the Hierarchy and pressing rename


Right now, your collectable will have a transform component, likely along with a Mesh Filter and Mesh Renderer. Those components will help Unity put the model into the 3D space.

We're going to add a new component to the collectable: a collider. Also known as a hitbox, this component detects when an object touches another object. In the Inspector for the collectable, click Add Component at the bottom

Physics → Box Collider

You'll see a green box around your object now, this is your collider

In the collider's settings (in the Inspector), we're going to click the box that says Is Trigger.

When testing, the cursor will be locked. To unlock it so you can stop or pause the game, press Escape

There are two types of colliders in Unity - normal colliders and trigger colliders. In a trigger collider, the two objects register that they touch but they don't actually collide (they can go through each other without slowing down).

In Mario games, for example, if you slowed down every time you touched a coin it would be a real pain.

Jumping into scripts

We've done a lot in the Unity interface, but in order to do some things we have to use coding. The language we're using is called C# (pronounced “c sharp”). It's based on C just like C++ is, if you've heard of that. We're going to write scripts that are attached as components to our objects!

The characters that we dragged in from the Standard Assets already have some scripts on them. We've been editing the variables in those scripts to do things like change the characters' speeds. You can see these scripts in the Inspector.

But to actually write a script, we need to use a text editor. We're going to create our first script using a text editor called Atom or Visual Studio! (You may need to create a Microsoft account to use Visual Studio) Go to the collectable and click the Add Component button in the Inspector. Click New Script and name it CollectableScript. Make sure the language is set to C#.

Click Create and Add. It will show up in the Inspector. Double-click on the box that says CollectableScript to open it in the text editor.


Our First Script!

This is a default C# script in Unity. For now, we can ignore the first few lines. These are called namespaces and we will use them in a bit. The code we'll be typing begins on line 5. Every time we make a script in Unity, it makes a class with that script name. All of our code will take place in that class between the opening bracket on line 5 and the closing bracket on line 16.

Each script also comes with two default functions. A function is a group of code (between the curly brackets after the function name) that we can execute at certain times. Unity has a bunch of functions built-in, and we can also write our own!


Here are the two default functions:

Start function - code between the brackets will run once at the start.

Update function - code between the brackets will run once every frame. Games usually run at 30 or more frames per second, so this code is basically always running!

Anatomy of a function

A function has basically 5 parts:

Name - the name of the function!

Start - the bracket at the beginning of the function's code.

End - the bracket at the end of the function's code.

Parameters - things that the function needs to run its code.

Return type - what the function spits out when it is done running

Don't worry about parameters and return type just yet. Just understand that a function is a section of code between two brackets that runs at certain times. Clear some space for new functions by going after the bracket that ends the Update function and pressing enter a few times.
