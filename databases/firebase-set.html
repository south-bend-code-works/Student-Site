Firebase: Set

In this lesson you will:

Create your first database, and send data to it

What You Will Make

Firebase Realtime Database is Firebase’s original database product. It’s a cloud-hosted NoSQL database that can store and sync data in real time for the user. FIrebase is also a Google owned product. One basic way to add data to our Realtime Database is Firebase’s ‘set’ method, which we’ll be using in this lesson!

HTML

On your computer, create a new folder for your project. Give it a name that reminds you that it is a Firebase project.

Open your text editor and create a new HTML file.
Give your html file 3-4 input fields. We'll grab the data for our database from what the user enters into these.

Give each input field a different id, based on the type of information you want the user to put into it. For example, we could give an id of "name" to a field that we want the user to enter their name into.

Give each field placeholder text that also matches the information you'd like the user to enter into it

Create a button underneath your input fields. We'll use this to submit our data to the database.
Give the button an id, "submit" works fine.

Give the button a "type=button" attribute. This will stop the button from reloading the page when we click on it.

Everything in Javascript is considered to be an object.
As a result of this, we can create our own objects. We can even create objects that exist to shorthand our code! As an example of this, we're going to create a variable called "database"

We'll set database equal to "firebase.database()".
firebase.database is a built-in firebase method that we use to access our database through our JS. Now whenever we need to use it, we can just say "database" instead!

Next, we're going to create the function that'll send our data. Create a function called "setUserData".
Back outside of our setUserData function, we'll tie running the function to the submit button we made in our html through its ID.

Make variables for each input field you created, and set them equal to the value of your input field.
Example:

Once we've created our variables, we'll use our database variable to reference and set our data:
  • .ref is another method, and it’s short for reference. When we use it here, we’re essentially saying “here is my database, I want to reference it"

  • ‘users/’ is the path that we’ll want to access within our database. If the path we specified doesn’t exist, .set will create one that matches.

  • “+” signs in JS are used for concatenation. Concatenation is the joining of strings together. Here we’re concatenating a variable called ‘userName’ onto the end of our ‘users/’ path. This means that whenever a new user is made, an object with a name that matches their user name will be created with all their attributes under the ‘users’ path.

  • The red strings before the colons are the names our users’ attributes will have in the database. Note that each one is assigned to one of the variables that we made to track our input fields.

  • Separate each attribute with a comma, the comma after the final attribute is optional.
