Binding UI Actions

In this lesson you will:

Edit vive's default keybindings

What You Will Make

SteamVR for Unity comes with built in actions that you can use in your game like Teleport, Grab, Pinch, etc…
However, if we want to use the controller actions to do some other action in our game, we will need to bind this action to the particular function.
There are three things we need to do.
Sign into your Unity account to access your games.

Creating Actions

Open the SteamVR Input Window
Window Menu -> SteamVR Input

There are four built in action sets, default, platformer, buggy, and mixedreality. For now we will just edit the default set. Later on, you could create your own action set, or use a different built in set.

To add a new action, click the + button under the "In" list

You will need to name your action. Make sure to use a descriptive name.

There are six action types you can choose from:
  1. Boolean - an action that is on or off, like a button click
  2. Vector1 - an action that will return a single value, like how far the trigger is pulled
  3. Vector2 - an action that will return an x,y pair of values like a planar location
  4. Vector3 - an action that will return an x,y,z triple of values like a 3D location
  5. Pose - this action captures the pose that the user’s hand is in at the moment. Particular poses can be programmed for items depending on the shape and how the user should interact with them.
  6. Skeleton - this action captures the skeleton of the users hands, for use in hand/object animations. Depending on the controller/input device used, this can be very simple data or very complex data

For our purposes, we can keep the Required option set to suggested.
  • Mandatory means that this action must be bound to a user input for the game to compile and run.
  • Suggested means that this action should be bound to a user input for the game to run.
  • Optional means that this action is optional and won't affect gameplay.
Once you've created your action, be sure to press "Save and Generate!"
Binding the Actions to Controls

From inside the SteamVR Input window, open the binding UI

This will open the binding interface in your web browser.

Edit the vive_controller in Current Binding.

By default, the following bindings are in place, and are mirrored(the same on both controllers)
    Grip: Use as Button
      Click -> Grab Grip
    Menu and Skeleton are empty
    Trigger: Use as Button, Use as Button, and Use as Trigger
      Click -> Interact with UI
      Click - > Grab Pinch
      Pull -> Squeeze
    Trackpad: Use as Button
      Click -> Teleport

You can remove any of these bindings, replace them, or add your own as well.

Here are the inputs that each controller feature can accept.
Grip Button
 Boolean
 Vector1(but only 0 or 1, no intermediate values.)
Menu Button
 Boolean
 Vector1(see above)
Trigger
 Boolean
 Vector1(range of values)
Trackpad as Trackpad
 Click -> Boolean, or Vector1
 Touch -> Boolean, or Vector1
Trackpad as Dpad
 North, East, South, West, and Center all accept Boolean or Vector1
System Button
 Boolean
 Vector1
Getting Input from the Controls in a Script

Once you've created the Action in the SteamVR Input, and bound it to the appropriate controller action, you are ready to get values from that action!

You need to open whatever script you're using to get the input values.

Inside that script, you will declare a variable that will recieve the input from the controller.

The variable type will match whatever action type you defined.
For example, if you created a boolean, you will make your variable of type SteamVR_Action_Boolean.

Each type will be SteamVR_Action_******, where ****** is whatever type you've used.

For boolean actions, to get the value, you will call varName.GetStateDown(InputSource)
The input source can be a specific object, like a hand, or it can be set to Any
Use SteamVR_Input_Sources.Any to get input from any source.

For Vector2 actions, to get the value you will call varName.GetAxis(InputSource).

Then you can use the values you have to move your character, or perform some other action in the game.
