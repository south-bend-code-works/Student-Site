Getting Started

We start again with some updated starter code. As you peruse the code, you’ll notice a new file called environment.js that has a new class in it called Pointer.

Take a look at this class. See how we have extended Ball to take advantage of Ball.moveWithMouse()? We have a custom constructor() and draw() method to give us the behavior we want.

We are also translating the origin (0,0) to the center of the canvas requiring a bit of acrobatics to get right but will really pay off.

If you look in sketch.js, you’ll see that instead of moving toward the mouse pointer player.aim = createVector(mouseX,mouseY);, we are moving toward the position of pointer pointer.pos, though we have to offset it by our translation, so we are making a new vector. When you run this code, you’ll see the bouncing faces have been taken out to eliminate distraction. The code to bounce them around the canvas has also been removed, and since our bounds will extend past the edges of the screen, the keepInBounds() method in Ball has been renamed keepOnCanvas().

Finally, the biggest difference is that a momentum vector has been added that makes our guy slide around in a more fluid way.

Tasty Morsels

Now that we have a bit of food, let’s have a meal. We need to know when our player is grabbing a snack on the go — when it collides with a pellet. For a collision to happen, they need to touch, right?
Since everything here is a circle, let’s think about how we can tell when two circles are touching. The definition of a circle is just a continuous curve drawn at precisely one distance from a single point — that distance we call the radius, that point, the center. Our circle positions are the center. So no matter the sizes of the circles in question, if we add the radius of each one together and find that it’s greater than the distance from their centers, the circles must be overlapping.

What is the method we used last lesson that can tell us the distance between the two vectors? Pythagoras had something to do with it.
Hover for a while to reveal a hint
It’s p5.Vector.dist() which we can call like so distanceBetweenVectors = someVector.dist(otherVector).
Before we get to using this method, we need a new method to use it, one to detect a collision between the food pellet and the player. Since this will be useful for any ball and face, let’s put it in Ball.

Go to shapes.js and put isCollision(other) {} at the end of the Ball class.
Notice how we put is in the name of the method. This method is going to check whether it has collided with the other and just return a boolean value — true or false. Putting that is making it sound like a question in broken English is often how we name boolean functions.

So let’s return a value. We could check a condition to see if they are touching then return true if true, otherwise return false. But what if we just returned the result of the condition? In the body of isCollision(), put return condition;

Now we need to see if the distance between the centers is less than the sum of the two radiuses, which can get by adding sizes and dividing by two. In place of condition put the following: 

                  this.pos.dist(other.pos) <
                    (this.size + other.size) / 2;
(This is broken into two lines so it doesn’t have a hard-to-read line break.)
In computer programming, you’ll often hear of a wrapper function where the purpose of a function is just make another function easier to deal with without changing any calculations within it — kind of like a wrapper encloses a piece of candy. The candy is the same either way, but the one without the wrapper is often pretty gross. We can broaden that definition to think of this method as a wrapper around our candy condition.

Now that we have our condition, let’s do something when they collide. Declare a new method called collidesWith() in Ball right beneath isCollision() and give it the parameter other.

Conspicuous Consumption
That takes care of the area, but what about this consumes() method: this.consumes(other) and other.consumes(this)? It also needs to be implemented.

Declare a new consumes() method in Ball right beneath collidesWith(). Give it one parameter called other.

In the body of the method, copy and paste this code: 

            let diffArea = this.area - other.area;
            diffArea = min(diffArea, other.area);

            this.area += diffArea;
            other.area -= diffArea;

            this.size = circleAreaToSize(this.area);
            other.size = circleAreaToSize(other.area);
All that this code does is find the difference in sizes, checking with min() that the difference is not greater than the one being eaten, and adding it to the eater.
